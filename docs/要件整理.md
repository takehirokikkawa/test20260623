## 1. 課題概要
技術記事データを基盤としたナレッジマネジメントシステム「TechInsight」を構築する。
評価対象は (a) 基本的なデータ管理（CRUD）、(b) AIによる**セマンティック検索**の統合、(c) 実用的なシステム設計能力。

- 形式: フルスタック実装課題
- 提出: **public リポジトリ**（README 必須）＋ 簡易DB設計書 ＋ 簡易API設計書 ＋ 実装説明（後述の観点別）
- 再現要件: **評価者のローカルPCで完全再現**できること

## 2. 技術要件（指定）
| 層 | 指定 |
|----|------|
| Backend | Python / FastAPI |
| Frontend | Next.js / TypeScript / React |
| Database | PostgreSQL |
| AI/ML | 任意モデルでのベクトル化（Embedding）実装 |
| Infra | **Docker Compose でワンコマンド起動**（`docker compose up` でマイグレーション→Web起動まで） |

## 3. 実装タスク
### タスク1: データマイグレーションと環境構築
- 提供 `articles.csv`（1,000行）を読み込みDBへ保存するスクリプト/プロセス。
- **実データは1万件規模に増える前提**でクエリ効率を考慮した設計・実装。
- Docker Compose で DB / API / Frontend を連携起動。`docker compose up` でマイグレーション実行＆Web起動まで完結。

### タスク2: Backend API
- CRUD API: 記事の取得（検索含む）/ 新規作成 / 更新 / 削除。
- 検索API: 自然言語クエリに対し記事本文との**意味的類似度**で検索結果を返す。モデル・類似度計算・検索ロジックは自由。

### タスク3: Frontend
- 記事一覧・検索画面（キーワード or セマンティック検索）。
- 記事詳細表示（モーダル等で全文確認）。
- 管理機能（投稿・編集・削除のフォーム/モーダル）。

## 6. 設計判断（提案 / 要確認）

### 6-1. Embedding は「ローカル完結」が必須
評価者はAPIキーを持たない前提のため、OpenAI等の外部Embedding APIに依存すると再現不能になる。
→ **`sentence-transformers`（多言語: `intfloat/multilingual-e5-small` 等）をAPIコンテナ内で実行**する方針を提案。
   - モデルはビルド時にDockerイメージへ同梱（初回起動時にネットワーク不要 or 初回のみpull）。
   - 外部API不要・キー不要・完全ローカルで再現可能。

### 6-2. ベクトル検索基盤
- **PostgreSQL + `pgvector` 拡張**を採用（DB指定がPostgreSQLのため、別ベクトルDBを足さず一貫性を保つ）。
- 1万件規模 → `ivfflat` or `hnsw` インデックスで近似最近傍検索。
- キーワード検索は PostgreSQL 全文検索（GIN）併用 → ハイブリッド検索も検討。

### 6-3. スケーラビリティ（1万件前提）
- 一覧はカーソル/オフセットページネーション。
- Embedding生成はマイグレーション時にバッチ処理。
- インデックス設計を明示（後述DB設計書）。

### 6-4. アーキテクチャ（チーム開発意識）
- monorepo: `backend/`（FastAPI, レイヤード: api/service/repository）, `frontend/`（Next.js App Router）, `db/`（マイグレーション）, `docker-compose.yml`。
- 型・スキーマ共有、Lint/Format、テスト、env サンプルを整備。
